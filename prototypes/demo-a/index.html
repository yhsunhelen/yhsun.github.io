Phone Holder for GarageBand

A physical-key add-on for playing GarageBand piano on a phone—designed to bring a more piano-like posture and press feedback to touch-screen performance.

Overview

Problem

Playing piano on a phone lacks the posture, press feel, and control of a real keyboard.

Goal

Create a physical phone holder that reliably triggers the touch screen and provides a satisfying return feel.

Outcome

Iterated from sketches to hand-built mechanisms to a 3D-printed medium-fidelity prototype.

Portfolio one-liner

Turned touch-screen tapping into a physical keypress using conductive triggering and spring return, then used a medium-fidelity build to uncover coupling issues and define the next iteration.

Key Decisions

Key mapping: not 1:1

A strict 1:1 mapping to on-screen keys would make the physical keys too narrow. Using a pentatonic-inspired layout allows wider keys and better playability.

Mechanism selection

Explored bell-crank and push-slide-rail concepts; hands-on tests informed which approach was more practical for buildability and motion clarity.

Touch triggering

Tested conductive materials (wire, wetted cord, felt, etc.) to trigger capacitive touch without direct finger contact. Copper foil became the more controllable contact point in later builds.

Iteration driven by failure

Spring tilt, unstable linkages, and axis coupling were treated as signals, and each issue led toa structural redesign (spring anchoring, joint geometry, and axis strategy).

Process

1

Sketching: form + motion

Sketched key layouts and motion paths in parallel, comparing mechanism options and planning constraints for thickness, travel, and contact placement.

  • Wider keys by avoiding strict 1:1 mapping
  • Mechanism concepts prepared for physical validation
2

Hand-built tests: validate motion

Built simplified mechanism mockups to verify smooth motion and feasibility, then used quick volume models to confirm overall size and phone fit.

  • Physical testing guided mechanism choice
  • Volume checks ensured device fit and ergonomics
3

3D-printed iterations: trigger + return + joints

Iterated on conductive touch triggering and spring return. Addressed spring misalignment and unreliable linkages by refining anchor points and rebuilding joints for stability.

  • Conductive contact strategy matured
  • Spring return tuned for consistent reset
  • Joints redesigned for robustness
4

Medium-fidelity build: integration learning

Assembled a medium-fidelity version (base + 7-key module). Used pivots and copper-foil tips for capacitive touch. Discovered axis coupling: pressing one key could influence others—highlighting the next iteration focus.

  • Module assembly validated
  • Copper foil improved touch consistency
  • Axis coupling became a key next-step problem
